# DESIGN SYSTEM UPGRADE

Now let’s talk raw UI.

## 1. Typography

Your app needs stronger hierarchy.

You need clear levels:

* hero scores
* page titles
* section headers
* card titles
* metadata
* tiny labels

Right now it sounds like the visual system is probably too close in weight.

Use bigger contrast:

* bold for headlines
* medium for key labels
* muted for metadata
* very restrained tertiary text

## 2. Card variety

Right now it probably uses too many similar dark rounded cards.

You need multiple card families:

* hero cards
* score rows
* article cards
* cluster cards
* insight modules
* standings rows
* player mini cards

That variety creates rhythm.

## 3. Color logic

Your current palette is clean:

* near-black
* charcoal
* orange
* red live badge

Good foundation.

Now add system roles:

* orange = brand/action
* red = live/urgent
* green = up/win/positive trend
* blue = neutral/information
* purple = AI/intelligence mode
* yellow = caution/watch

That makes the product smarter visually.

## 4. Depth

Use more depth subtly:

* top-level page backgrounds
* layered cards
* soft inner highlights
* glassy overlays very lightly
* glows only when meaningful

Not gamer UI.
Just premium tension.

## 5. Motion

This matters.

Motion should communicate state:

* live pulse
* score change flash
* article mode crossfade
* standings row expand
* chip selection slide
* hero card auto-cycle carefully
* pull-to-refresh with brand identity

This makes it feel built, not assembled.
