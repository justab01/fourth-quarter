I’ve added a new module and database schema to help you fetch and store real‑world tennis, fighting and track‑and‑field data for your sports app.

### What’s included

**1. Comprehensive data helper file (`data_sources.js`)** – This Node module shows how to fetch both live and historical data from free sources and open datasets:

* **TheSportsDB API** – functions like `fetchUpcomingEvents`, `fetchRecentEvents`, `fetchEventDetails` and `searchTennisPlayer` pull current and past events for the ATP and WTA tennis tours (league IDs 1155 and 1156) and the UFC (league ID 4443). TheSportsDB offers a free tier via API key “123”, but you can use your own key in the `TSD_API_KEY` environment variable.
* **Historical tennis datasets** – `loadCsvFile` reads CSV files from Jeff Sackmann’s tennis repositories, which contain decades of ATP and WTA results and rankings. The README notes that these datasets are licensed under CC BY‑NC‑SA 4.0 and provide rankings and match stats from 1985 onwards.
* **World Athletics GraphQL API** – functions like `searchCompetitors` and `getCompetitorResultsByDiscipline` interact with World Athletics’ GraphQL backend, the same API used by their website. The open‑source “World‑Athletics‑data‑collection” project documents the endpoint and key, and the module demonstrates how to query it for athletes and their results by discipline. The API key can change, so the code allows you to set `WA_ENDPOINT` and `WA_API_KEY` via environment variables.

You can import this module wherever you need to seed your database or expose live data through your server. The code is well‑commented to explain how to use each function.

**2. Extended database schema (`db_tennis_mma_athletics.sql`)** – This SQL script adds tables for individual sports: `athletes`, `tournaments`, `matches`, `events` and `performances`. It also inserts three new sports—Tennis, Mixed Martial Arts and Athletics—into the `sports` table. The schema lets you store tournament details, head‑to‑head matches (for tennis or MMA), track‑and‑field events and athlete performances. Running this script after the original `db.sql` will create the necessary tables in SQLite.

### How to use

1. **Install dependencies:** the helper uses `axios` for HTTP requests and `csv-parser` for reading CSVs. Install them with `npm install axios csv-parser`.
2. **Fetch live data:** call `fetchUpcomingEvents(1155)` or `fetchRecentEvents(1156)` to get ATP and WTA events, or `fetchUpcomingEvents(4443)` for UFC cards. For individual athletes, call `searchTennisPlayer('Novak Djokovic')`.
3. **Import historical data:** download the CSV files from Jeff Sackmann’s repositories and use `loadCsvFile()` to parse them into objects. You can then insert these rows into your SQLite tables.
4. **Get athletics results:** use `searchCompetitors({ query: 'Bolt' })` to find an athlete’s `aaAthleteId` and then pass that ID to `getCompetitorResultsByDiscipline()` with a discipline code (e.g. `"100m"`). Remember that the World Athletics API key may change and should only be used in compliance with their terms.
