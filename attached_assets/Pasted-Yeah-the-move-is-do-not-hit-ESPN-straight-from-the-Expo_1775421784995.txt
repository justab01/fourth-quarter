Yeah — the move is **do not hit ESPN straight from the Expo client**. Put a **sports data layer in your Express API**, normalize the responses there, cache them, then let the app read one clean internal API.

That’s the safest version of what you’re trying to do because ESPN’s public endpoints are **unofficial**, can change, and should be wrapped with caching/error handling. The endpoint families most people use are on `site.api.espn.com` and `sports.core.api.espn.com`, and community docs specifically warn they’re unofficial and best used behind your own service layer. ([GitHub][1])

For your setup, I’d use this stack:

* **Primary source:** ESPN public web/site endpoints for scores, schedules, standings, summaries, rosters
* **Optional helper:** `sportsdataverse` for some leagues/workflows in Node
* **Fallback source:** TheSportsDB free tier for missing artwork/basic event data, but know their free tier is limited and some newer/livescore methods sit behind premium/v2 access. ([TheSportsDB][2])

Below is the code path I’d actually put in your app.

## 5) Build the normalizer

This is the key. Your mobile app should not care what ESPN’s raw JSON looks like.

### `src/services/normalizeScoreboard.ts`

```ts
export type NormalizedCompetitor = {
  id: string;
  teamId?: string;
  displayName: string;
  shortName?: string;
  abbreviation?: string;
  logo?: string;
  score?: string;
  winner?: boolean;
  homeAway?: 'home' | 'away';
  recordSummary?: string;
};

export type NormalizedEvent = {
  id: string;
  name: string;
  shortName?: string;
  date: string;
  status: {
    state?: string;
    detail?: string;
    shortDetail?: string;
    completed?: boolean;
  };
  venue?: string;
  league?: string;
  seasonType?: number;
  competitors: NormalizedCompetitor[];
  situation?: any;
  links?: any[];
};

export function normalizeScoreboardEvent(event: any): NormalizedEvent {
  const competition = event?.competitions?.[0];

  return {
    id: String(event?.id ?? ''),
    name: event?.name ?? '',
    shortName: event?.shortName ?? '',
    date: event?.date ?? '',
    status: {
      state: competition?.status?.type?.state,
      detail: competition?.status?.type?.detail,
      shortDetail: competition?.status?.type?.shortDetail,
      completed: competition?.status?.type?.completed,
    },
    venue: competition?.venue?.fullName,
    league: event?.league?.name,
    seasonType: event?.season?.type?.type,
    competitors: (competition?.competitors ?? []).map((c: any) => ({
      id: String(c?.id ?? ''),
      teamId: c?.team?.id ? String(c.team.id) : undefined,
      displayName: c?.team?.displayName ?? c?.team?.name ?? 'Unknown Team',
      shortName: c?.team?.shortDisplayName,
      abbreviation: c?.team?.abbreviation,
      logo: c?.team?.logo || c?.team?.logos?.[0]?.href,
      score: c?.score,
      winner: c?.winner,
      homeAway: c?.homeAway,
      recordSummary: c?.records?.[0]?.summary,
    })),
    situation: competition?.situation ?? null,
    links: event?.links ?? [],
  };
}
```

## 6) Create the scoreboard service

### `src/services/espnScoreboardService.ts`

```ts
import { espnSiteApi } from '../lib/espnClient';
import { sportsCache, getCacheKey } from '../lib/cache';
import { ESPN_LEAGUES, LeagueKey } from '../config/espnLeagues';
import { normalizeScoreboardEvent } from './normalizeScoreboard';

type ScoreboardOptions = {
  leagueKey: LeagueKey;
  dates?: string;      // YYYYMMDD
  groups?: string;     // for college filtering if needed
  limit?: number;
};

export async function getScoreboard({
  leagueKey,
  dates,
  groups,
  limit = 100,
}: ScoreboardOptions) {
  const leagueInfo = ESPN_LEAGUES[leagueKey];
  if (!leagueInfo) {
    throw new Error(`Unsupported league: ${leagueKey}`);
  }

  const cacheKey = getCacheKey(['scoreboard', leagueKey, dates, groups, limit]);
  const cached = sportsCache.get(cacheKey);
  if (cached) return cached;

  const { sport, league } = leagueInfo;

  const response = await espnSiteApi.get(`/${sport}/${league}/scoreboard`, {
    params: {
      dates,
      groups,
      limit,
    },
  });

  const data = response.data;

  const normalized = {
    league: data?.leagues?.[0]?.name ?? leagueKey,
    season: data?.season ?? null,
    day: data?.day ?? null,
    events: (data?.events ?? []).map(normalizeScoreboardEvent),
  };

  // tighter cache for live scoreboards
  sportsCache.set(cacheKey, normalized, 20);

  return normalized;
}
```

export async function getGameSummary(leagueKey: LeagueKey, eventId: string) {

## 8) Expose your own API routes

### `src/routes/sports.ts`

```ts
import { Router } from 'express';
import { getScoreboard } from '../services/espnScoreboardService';
import { getStandings } from '../services/espnStandingsService';
import { getGameSummary } from '../services/espnSummaryService';

const router = Router();

router.get('/scoreboard/:leagueKey', async (req, res) => {
  try {
    const { leagueKey } = req.params as { leagueKey: any };
    const { dates, groups, limit } = req.query;

    const data = await getScoreboard({
      leagueKey,
      dates: dates as string | undefined,
      groups: groups as string | undefined,
      limit: limit ? Number(limit) : 100,
    });

    res.json(data);
  } catch (error: any) {
    res.status(500).json({
      error: 'Failed to fetch scoreboard',
      message: error.message,
    });
  }
});

router.get('/standings/:leagueKey', async (req, res) => {
  try {
    const { leagueKey } = req.params as { leagueKey: any };
    const data = await getStandings(leagueKey);
    res.json(data);
  } catch (error: any) {
    res.status(500).json({
      error: 'Failed to fetch standings',
      message: error.message,
    });
  }
});

router.get('/summary/:leagueKey/:eventId', async (req, res) => {
  try {
    const { leagueKey, eventId } = req.params as { leagueKey: any; eventId: string };
    const data = await getGameSummary(leagueKey, eventId);
    res.json(data);
  } catch (error: any) {
    res.status(500).json({
      error: 'Failed to fetch game summary',
      message: error.message,
    });
  }
});

export default router;
```

## 9) Use it in your Expo app

### `src/api/sports.ts`

```ts
const API_BASE = 'https://your-api-domain.com/api/sports';

export async function fetchScoreboard(leagueKey: string, dates?: string) {
  const url = new URL(`${API_BASE}/scoreboard/${leagueKey}`);
  if (dates) url.searchParams.set('dates', dates);

  const res = await fetch(url.toString());
  if (!res.ok) throw new Error('Failed to load scoreboard');
  return res.json();
}

export async function fetchStandings(leagueKey: string) {
  const res = await fetch(`${API_BASE}/standings/${leagueKey}`);
  if (!res.ok) throw new Error('Failed to load standings');
  return res.json();
}

export async function fetchSummary(leagueKey: string, eventId: string) {
  const res = await fetch(`${API_BASE}/summary/${leagueKey}/${eventId}`);
  if (!res.ok) throw new Error('Failed to load summary');
  return res.json();
}
```

### example hook

```ts
import { useEffect, useState } from 'react';
import { fetchScoreboard } from '../api/sports';

export function useScoreboard(leagueKey: string, dates?: string) {
  const [data, setData] = useState<any>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  async function load() {
    try {
      setLoading(true);
      setError(null);
      const result = await fetchScoreboard(leagueKey, dates);
      setData(result);
    } catch (e: any) {
      setError(e.message || 'Unknown error');
    } finally {
      setLoading(false);
    }
  }

  useEffect(() => {
    load();
  }, [leagueKey, dates]);

  return { data, loading, error, reload: load };
}
```

## 10) Add multi-sport “home feed” aggregation

This is where the app starts feeling like a real product.

### `src/services/homeFeedService.ts`

```ts
import pLimit from 'p-limit';
import { getScoreboard } from './espnScoreboardService';
import type { LeagueKey } from '../config/espnLeagues';

const limit = pLimit(4);

export async function getHomeFeed(leagues: LeagueKey[], dates?: string) {
  const results = await Promise.all(
    leagues.map((leagueKey) =>
      limit(async () => {
        try {
          const scoreboard = await getScoreboard({ leagueKey, dates });
          return {
            leagueKey,
            success: true,
            data: scoreboard,
          };
        } catch (error: any) {
          return {
            leagueKey,
            success: false,
            error: error.message,
          };
        }
      })
    )
  );

  return results;
}
```

## 11) What to use for each screen

This is the cleanest split:

**Use ESPN scoreboard**

* Home
* Live
* League scoreboard
* Top strip / today games
* basic game cards

**Use ESPN summary**

* Game detail
* leaders
* team stats
* drives / plays / events where available
* injuries / odds / broadcasts when present

**Use standings**

* division/conference tables
* season records
* playoff race blocks

**Use roster/team endpoints later**

* team page
* player page
* season leaders
* team schedule

The public ESPN docs gathered by the dev community show these endpoint families covering scores, teams, standings, athletes, and detailed data, but also note there’s no official support and you need your own wrapper/caching strategy. ([GitHub][1])

## 12) About your “out of scope” part

Your note is basically right:

* **Paid feeds**: out
* **True live shot-by-shot or universal high-granularity event tracking across every sport**: not reliable from free public ESPN endpoints
* **Some sports/leagues do have richer play data than others**
* **Basketball/football/hockey coverage is usually way better than niche sports**
* **SportsDataverse explicitly documents play-by-play support only for certain leagues/sports, not everything under the sun** ([SportsDataverse][4])

So your backend should be built with **capability flags**:

```ts
export const LEAGUE_CAPABILITIES = {
  nba: { scoreboard: true, summary: true, standings: true, pbp: true },
  nfl: { scoreboard: true, summary: true, standings: true, pbp: true },
  nhl: { scoreboard: true, summary: true, standings: true, pbp: true },
  mlb: { scoreboard: true, summary: true, standings: true, pbp: 'limited' },
  mls: { scoreboard: true, summary: true, standings: true, pbp: 'limited' },
  epl: { scoreboard: true, summary: true, standings: true, pbp: 'limited' },
  'college-football': { scoreboard: true, summary: true, standings: true, pbp: true },
  'mens-college-basketball': { scoreboard: true, summary: true, standings: true, pbp: true },
};
```

That way the UI does not promise what the data cannot deliver.
