UNIVERSAL GAME LOG ARCHITECTURE

The core principle

Every sport should follow the same master logic:

Season / Year → Competition Context → Stage / Segment → Individual Event / Game

That means no more dumping all stats into one flat page.

The page should always answer, in this order:
	1.	What season or year am I viewing?
	2.	What type of competition was this?
	3.	What stage of that competition am I in?
	4.	What happened in each game, match, fight, race, or event?

That gives you consistency across sports while still letting each sport feel native.

1. MASTER UNIVERSAL STRUCTURE

Level 1 — Season / Year selector

Top dropdown:
	•	2025–26
	•	2024–25
	•	Career
	•	All Seasons

This exists for almost every sport.

For sports without traditional seasons, use:
	•	2026
	•	2025
	•	Career

Examples:
	•	NBA: 2025–26
	•	NFL: 2025
	•	Tennis: 2026
	•	Golf: 2026
	•	UFC: 2026

⸻

Level 2 — Competition Context

Every event gets tagged into one of these buckets depending on sport:
	•	Preseason / Exhibition
	•	Regular Season / League Play
	•	Tournament / Cup
	•	Play-In / Qualification
	•	Playoffs / Postseason / Knockout
	•	Championship / Finals
	•	All-Star / Showcase
	•	International
	•	Summer League / Developmental
	•	Special Event

Not every sport uses every bucket, but the framework stays the same.

⸻

Level 3 — Stage / Segment grouping

Inside each context, logs are grouped differently depending on sport.

Examples
	•	NBA Regular Season → by month
	•	NBA Playoffs → by round
	•	NFL Playoffs → Wild Card / Divisional / Conference Championship / Super Bowl
	•	Tennis tournament → Round of 128 / 64 / 32 / 16 / QF / SF / Final
	•	Golf tournament → Round 1 / 2 / 3 / 4
	•	Track meet → Heats / Semis / Final
	•	UFC → Event card / fight night

This is the key.
Regular season gets grouped by time. Tournament/postseason gets grouped by bracket stage.

⸻

Level 4 — Individual row

Each row is the actual game/match/event log entry.

Each row should always have:
	•	date
	•	opponent / event / field
	•	result / placement
	•	core stats / performance summary
	•	expandable detail

4. UNIVERSAL GROUPING RULES

This is the logic engine.

Rule A — Regular season or league play

Group by:
	•	month for basketball/baseball/hockey/soccer
	•	week for football
	•	tournament chronology for tennis/golf/combat
	•	meet for track

Rule B — Bracket / postseason / knockout

Group by stage:
	•	Round 1
	•	Quarterfinal
	•	Semifinal
	•	Final
or sport-specific equivalents

Rule C — Single-event sports

Group by event:
	•	UFC 317

Never flatten championship runs into generic rows.

Use:

	•	champion badge
	•	finals summary card
	•	title defense badge
	•	podium badge
	•	record tags

16. COPY-PASTE DEV HANDOFF

Use this exactly:

Universal Game Log Architecture Requirements
	•	All athlete game logs must use a hierarchical structure: Season/Year → Competition Context → Stage/Segment → Individual Event Row
	•	Competition contexts must support categories such as Preseason, Regular Season, Tournament/Cup, Play-In/Qualification, Playoffs/Postseason, Championship/Finals, All-Star/Showcase, International, and Special Events
	•	Regular season logs should group by time bucket appropriate to the sport, such as month for basketball/baseball/hockey or week for football
	•	Tournament and postseason logs should group by stage or round, not by month
	•	Single-event sports such as tennis, golf, combat sports, and track must group logs by tournament/meet/event first, then by round/session/stage
	•	Every competition section should include a summary card showing games/events played and core averages or performance summaries
	•	Championship runs, finals appearances, podium finishes, and title wins must be visually preserved with badges, trophy markers, and dedicated championship summary sections
	•	Game rows must use a mobile-first layout with top-line context and a second-line quick stat summary
	•	Rows must support expandable details or bottom sheet detail view
	•	Row stat templates must be role-aware when needed, including QB/RB/WR/DEF/K for football, hitter/pitcher for baseball, skater/goalie for hockey, goalkeeper/outfield for soccer
	•	The system must support normalized tagging for competition type, stage name, event grouping, result type, achievement badges, and advancement/elimination status
	•	Career view must support year-by-year expansion with major achievements surfaced at the season header level
	•	Filters must dynamically update both visible rows and summary cards based on current view selection
