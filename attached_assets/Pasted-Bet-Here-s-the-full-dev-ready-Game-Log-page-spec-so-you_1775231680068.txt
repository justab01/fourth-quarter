GAME LOG PAGE — FULL MOBILE REDESIGN SPEC

Core goal

Build a mobile-first, scan-fast, premium-feeling Game Log page for athlete profiles that feels:
	•	cleaner than ESPN
	•	less clunky than a desktop table
	•	better organized by month
	•	faster to read in one thumb
	•	expandable when users want more depth

This page should prioritize:
	•	instant recognition
	•	strong spacing
	•	clean opponent identity
	•	monthly grouping
	•	compact stat digestion
	•	deeper detail on tap

⸻

1. PAGE ARCHITECTURE

Overall structure

From top to bottom:

1. Sticky compact player header

2. Sticky horizontal tab bar

3. Page intro block

4. Filter / season controls

5. Monthly game log sections

6. Expandable game details or bottom sheet

7. Optional persistent quick actions

⸻

2. HEADER REDESIGN

Do not use the giant hero banner on Game Log

That big player image section is fine for Overview, but on Game Log it wastes vertical space and makes the stat content feel buried.

Replace it with a compact sticky header

Compact player header contents

Left side:
	•	player headshot, 40–48px
	•	player name
	•	team name + league
	•	position badge

Right side:
	•	follow button
	•	maybe jersey number chip

Compact header height
	•	Expanded: 88px
	•	Collapsed sticky: 64px

Visual rules
	•	keep background tied to team color gradient, but much more subtle
	•	use a darker overlay for readability
	•	do not allow the art to dominate the screen
	•	remove oversized background jersey number from this tab

Example structure

Pacers • NBA
[GUARD]

with follow button on right

13. ROW VISUAL STYLE

Spacing

	•	vertical padding: 12–14
	•	row gap: 6–8 between rows if cards
	•	divider if flat-list style

Card or flat?

Best answer:
	•	use flat rows inside a month section

	•	do not over-card everything or it’ll get noisy

Press state

	•	expand details or open bottom sheet

14. EXPANDED GAME DETAILS

When the user taps a row, give them richer info.

Two options

Option A: Inline expansion

Row expands downward beneath itself.

Pros:
	•	fast
	•	contextual
	•	easier for scan-heavy use

Option B: Bottom sheet

Tap row opens a bottom sheet with richer details.

Pros:
	•	feels more premium
	•	more room for advanced content
	•	works better if you want links, visuals, and charts

Best recommendation

Use inline expansion for quick stats.
Use bottom sheet for full game details if user taps “See more.”

18. COLOR SYSTEM

Keep dark mode, but clean it up.

Base surfaces
	•	page background: near-black
	•	section surfaces: slightly lifted charcoal
	•	borders: subtle cool gray with low opacity
	•	text primary: off-white
	•	text secondary: muted gray

Accent

Use team color strategically, not everywhere.

Team color should appear in:
	•	active underline
	•	follow button outline/fill maybe
	•	subtle month average accent
	•	selected filters
	•	maybe thin left-edge highlight on rows

Do not flood the page with orange or purple or whatever. ESPN’s strength is restraint.

24. EMPTY / ERROR / LOADING STATES

Loading state

Do not show ugly spinners only.

Use skeleton loaders for:
	•	compact header
	•	month average strip
	•	5–8 fake rows

Empty state

Examples:
	•	No games found for this filter
	•	This player did not appear in any playoff games this season

Error state

Simple retry block:
	•	Couldn’t load game logs
	•	Retry button

Keep the visual language consistent.

Since this page can get long:

Must use
	•	virtualized list
	•	memoized rows
	•	image caching for logos/headshots
	•	deferred expansion content load if needed

Avoid
	•	rendering all expanded row content at once
	•	heavy shadows everywhere
	•	overly large logo assets
	•	giant hero images on stat tabs

Performance is part of why clunky feels clunky.
