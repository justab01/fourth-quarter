	•	easy to open

The main product truth

The app should have 3 layers:

1. Fast Layer

For the user who opens the app and wants:
	•	scores
	•	status
	•	who won
	•	who’s live
	•	what matters

2. Smart Layer

For the user who wants:
	•	averages
	•	last 5 games
	•	trends
	•	matchup context
	•	standings impact
	•	why this matters

3. Fan Layer

For the user who wants:
	•	reactions
	•	community pulse
	•	predictions
	•	shared takes
	•	fandom energy

The mistake would be making all 3 layers scream at once.

So the right move is:

keep the default experience clean, and let the deeper layers open only when wanted

What “friendly” means here

When you say friendly, that does not mean dumbed down.

It means:
	•	easy to scan
	•	no ugly overload
	•	no confusing stat walls by default
	•	no feeling like you need to be a gambling addict or spreadsheet nerd to use it
	•	but still deep enough for real fans

So the app should always show:
	•	main info first
	•	useful extra info second
	•	advanced info third

That’s the whole design principle.

Every major object in the app should follow this same pattern:

Level 1 — Snapshot

The quick answer

Level 2 — Context

Level 3 — Deep Dive

Stats, trends, charts, logs, advanced info

That should apply to:
	•	games
	•	teams
	•	players
	•	standings
	•	articles
	•	community posts

If you do that, the app stays clean.

What should be on a game page

A game page should not just be score + play-by-play.

It should feel like a game center.

Top layer
	•	score
	•	quarter/inning/period/set
	•	live status
	•	possession / runners on base / down & distance / serve / round
	•	venue
	•	watch/follow button

Next layer
	•	top performers
	•	current run / momentum
	•	why this game matters
	•	quick stat leaders

Then tabs
	•	Summary
	•	Plays
	•	Stats
	•	Trends
	•	Lineups
	•	Fan Pulse

That’s where you can sneak in the deep stuff without clutter.

Trends tab should include:
	•	team averages
	•	last 5 games
	•	head-to-head
	•	home/away form
	•	player trend lines
	•	season vs recent form

That’s the 365Scores/Sofascore utility layer you’re talking about.

⸻

How to do community without ruining the app

This is the hardest part.

You do not want to become a messy comment section app.

So do not make community the center.
Make it a layer.

Best way to do it:

Use structured community, not open chaos.

That means:

1. Fan Pulse

A lightweight reaction layer on games, teams, and players.

Examples:
	•	Who wins?
	•	Panic meter
	•	Player of the game vote
	•	Was that a foul?
	•	Confidence in this team?
	•	Is this team real?

That gives community feeling without turning into a sewer.

2. Quick Takes

Short fan reactions, not essay posts.

Think:
	•	one-line takes
	•	upvote/downvote or fire meter
	•	friends/followed users first
	•	team-specific feed

3. Polls

Very strong for sports.

Examples:
	•	Who closes this out?
	•	Better young core?
	•	Was it a bad call?
	•	Who should start next game?

4. Game Chat, but contained

Only inside a live game page.
Not global.
Not default-first.
Maybe behind a tab.

That keeps the rest of the app clean.

How to keep it holistic without too much

This is the real product discipline:

Every screen should ask:
	•	what’s the one thing the user came here for?
	•	what are the 2–3 most useful extra things?
	•	what deeper info should stay hidden behind a tap?

That’s it.

For example:

On a player page

Came for:
	•	who is this player and how are they doing?

Show:
	•	headshot
	•	averages
	•	last 5
	•	game log preview

Hide deeper:
	•	splits
	•	advanced stats
	•	shot map
	•	trend graphs

On a game page

Came for:
	•	score and what’s happening

Show:
	•	score
	•	status
	•	top performers
	•	momentum

Hide deeper:
	•	full play log
	•	deep stat table
	•	lineup details

That’s how the app stays human.
