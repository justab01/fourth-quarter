```
<div class="debt-list">
  <div class="debt-item">
    <div class="area">Auth</div>
    <div>
      <div class="fix">Add user accounts (Supabase Auth or Clerk)</div>
      <div class="issue">Everything lives in AsyncStorage. Can't do cross-device sync, social features, or any server-side personalization. This is the infrastructure prerequisite for every community feature.</div>
    </div>
  </div>
  <div class="debt-item">
    <div class="area">Search</div>
    <div>
      <div class="fix">Move from static 2,284-line index to real search backend</div>
      <div class="issue">The local player/team index will miss newly signed players, trades, and international athletes. Build a lightweight search endpoint backed by the ESPN API that covers the long tail.</div>
    </div>
  </div>
  <div class="debt-item">
    <div class="area">ESPN API</div>
    <div>
      <div class="fix">Add rate limit handling + fallback caching layer</div>
      <div class="issue">If ESPN's unofficial API has an outage or rate limits your server, the whole app breaks. Build a Redis or Supabase cache layer that serves the last-known good data while the API is down.</div>
    </div>
  </div>
  <div class="debt-item">
    <div class="area">AI Costs</div>
    <div>
      <div class="fix">Add cost controls for AI features (In One Breath, Article Modes, Live Tracker)</div>
      <div class="issue">Every AI call costs money. Once you have real users, "In One Breath" refreshing on every pull-to-refresh could get expensive fast. Add smart caching: only regenerate if the underlying data has changed since last generation.</div>
    </div>
  </div>
  <div class="debt-item">
    <div class="area">Analytics</div>
    <div>
      <div class="fix">Add basic event tracking (Mixpanel or PostHog free tier)</div>
      <div class="issue">Right now you have no idea which features users actually use, which tabs get visited, or where people drop off. You need this data to make smart product decisions in month 2.</div>
    </div>
  </div>
  <div class="debt-item">
    <div class="area">WebSocket</div>
    <div>
      <div class="fix">Stress test WebSocket connections for multiple simultaneous live games</div>
      <div class="issue">On a Sunday in NFL season, you could have 14 games live simultaneously. Verify the WebSocket architecture handles multiple concurrent game subscriptions without memory leaks or missed updates.</div>
```
