Season selector

Conference Finals

Dev rule you need to set

Tell your system this:

Game logs must support hierarchical segmentation by season, competition type, and stage. Regular season views should group by month. Postseason and tournament views should group by round or stage. Championship outcomes, play-in appearances, and tournament runs must be visually preserved, not flattened into generic logs.

That is the product rule.

Copy-paste handoff for dev

Use this exactly:

Expanded Game Log Segmentation Requirements
	•	Game logs must be organized by season first
	•	Within each season, logs must be segmented by competition context, not just dumped into one list
	•	NBA-supported competition segments should include: Preseason, Regular Season, NBA Cup, Play-In Tournament, NBA Playoffs, and Finals/Championship summary
	•	Regular Season logs should be grouped by month with monthly averages
	•	Playoff logs should be grouped by round, not by month
	•	Supported playoff round labels should include: First Round, Conference Semifinals, Conference Finals, NBA Finals

	•	Championship runs must be visually marked with badges such as Champion, Finals MVP, or Conference Champion when applicable
	•	Each competition section should include a summary header with games played and core averages
	•	Each stage or round should include game rows beneath it
	•	Game rows should preserve mobile-first formatting with date, opponent logo, result badge, and quick stats
	•	System must support universal adaptation of this same hierarchy for other sports, using months for regular season and rounds/stages for tournaments or playoffs
