Yes — draft needs to be a real system in the app, not some random tab you tack on at the end.

The clean answer is this:

Build a universal “Draft Center” layer across the app, then let each sport plug into it differently.

That works because several major leagues you care about already have formal draft ecosystems today: the NFL has a seven-round draft with 257 picks in 2026, the NBA has a draft plus lottery and combine flow, MLB runs its First-Year Player Draft during All-Star Week, the NHL holds its draft in late June, the WNBA has both its annual draft and, in 2026, an expansion draft, and MLS still runs the SuperDraft.  ￼

What to add to the app

You need a new product surface:

Draft Center

This should live as a first-class feature, not buried.

Best placements:
	•	top-level from Home as a featured rail when draft season is active
	•	inside each league page as its own tab
	•	inside team pages
	•	inside player/prospect pages
	•	inside news/search so draft content is discoverable year-round

Main league tabs that should support Draft Center now

Launch these first:
	•	NFL Draft
	•	NBA Draft
	•	WNBA Draft
	•	MLB Draft
	•	NHL Draft
	•	MLS SuperDraft

That is the real core set right now based on active official league draft structures.  ￼

Also support these special draft modes

Not just normal amateur drafts:
	•	Lottery
	•	Mock Drafts
	•	Combine / measurables
	•	Draft order
	•	Tradeable picks
	•	Expansion Draft
	•	Draft history
	•	Prospect rankings / big boards

That matters because the NBA officially surfaces combine data and key draft-process dates, MLB has special draft-order rules like Competitive Balance picks, and the WNBA currently has a separate expansion draft flow in addition to the regular draft.  ￼

What Draft Center should contain

A. Overview screen

This is the front page for each league’s draft.

Show:
	•	next draft date
	•	location
	•	order
	•	top prospects
	•	latest news
	•	mock draft carousel
	•	trade watch
	•	recent movement

For example, the 2026 NFL Draft is April 23–25 in Pittsburgh, the WNBA Draft is April 13, the MLB Draft is July 11–12 in Philadelphia, and the 2026 NHL Draft is June 26–27 in Buffalo.  ￼

B. Draft Order screen

Must show:
	•	all picks by round
	•	pick owner
	•	original owner
	•	traded-from note
	•	compensatory / competitive balance / special pick types where relevant

This especially matters in the NFL and MLB because the NFL runs a full seven-round order and MLB includes Competitive Balance rounds with special trade/forfeiture rules.  ￼

C. Prospect Board screen

This is your big board.

Sort/filter by:
	•	overall
	•	position
	•	school/team/club
	•	age
	•	country
	•	measurements
	•	projected round
	•	team fit

For NBA and NHL especially, this should feel premium because official league ecosystems already center prospects and combines heavily.  ￼

D. Mock Draft screen

This is one of the biggest engagement pieces.

Have:
	•	app official mock
	•	community mock
	•	team-specific mock
	•	lottery simulation for NBA/WNBA where useful
	•	scenario mode after trades

E. Pick Tracker screen

Live draft night screen:
	•	pick number
	•	team on the clock
	•	selection
	•	trade alert
	•	available best prospects
	•	grade
	•	clip/highlight
	•	instant fit analysis

F. Draft History screen

For every league:
	•	by year
	•	by team
	•	by school
	•	by player
	•	by position

This gives the app historical gravity.

What to do for sports without a normal draft

This is where you keep the app smart instead of forcing fake draft tabs everywhere.

For leagues/sports that mainly use transfers, academies, signings, or qualification systems, do not force “Draft.” Use a parallel acquisition label like:
	•	Prospects
	•	Signings
	•	Recruitment
	•	Transfer Market
	•	Academy Pipeline
	•	Roster Building

So:
	•	soccer outside MLS = transfer market first
	•	combat sports = signed fights / contender pipeline
	•	golf/tennis/track = prospects / junior pipeline / rankings, not draft
	•	college sports = recruiting + portal, not pro-draft tab inside the college product

That keeps the app honest.

Best product structure

Add a universal acquisition rail

Not every sport should show the same word.

Use this logic:
	•	if league has draft → show Draft
	•	if league has no draft but has prospects → show Prospects
	•	if league is transfer-driven → show Transfers
	•	if league is academy/development driven → show Pipeline

That’s how you scale without making the UI dumb.

Copy-paste handoff for dev

Draft System Requirements
	•	Add a universal Draft Center framework to the app for leagues with active draft systems
	•	Draft-enabled leagues at launch should include NFL, NBA, WNBA, MLB, NHL, and MLS
	•	Each draft-enabled league page must include a Draft tab
	•	Draft tab must support subpages for Overview, Order, Prospects, Mock Drafts, Teams, Live Tracker, and History

	•	Prospect pages must support draft-specific profiles with measurements, rankings, scouting notes, projection range, and team fit
	•	Live draft mode must support on-the-clock state, pick announcements, best available players, and trade alerts
	•	System must support multiple draft types including annual amateur drafts, lotteries, combines, expansion drafts, and league-specific special pick structures
	•	Data model must support original pick owner, current pick owner, round, overall pick number, trade status, and league-specific special pick tags
	•	Sports without draft systems should not force a Draft tab and instead use alternate acquisition surfaces such as Prospects, Transfers, Recruitment, or Pipeline
	•	Search and news systems must support draft content as its own discoverable content type
