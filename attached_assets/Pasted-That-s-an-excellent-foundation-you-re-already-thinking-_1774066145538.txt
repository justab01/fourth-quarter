### How ESPN’s Gamecast Systems Are Designed

Every ESPN Gamecast—NBA, NFL, MLB, Soccer, etc.—is a *renderer for processed event streams*.

They all share one core design:

```
External Feed → Ingest Engine → Event Normalizer → State Tracker → Visualization Layer
```

Each sport just has a different renderer, logic module, and event taxonomy.

***

### What ESPN Receives Live

ESPN rarely collects data directly; it partners with league data providers like Sportradar, Stats Perform, and the leagues' proprietary APIs (NFL Next Gen Stats, MLB BAMTech, FIBA stats feeds, etc.). Each feed sends **atomic events** every few seconds:

- **Basketball:** shot attempts (location, result, assister), rebounds, turnovers, fouls, substitutions, time changes.  
- **Football:** play type, yard line, down & distance, passer, receiver, result, live clock.  
- **Baseball:** pitch type, speed, contact result, baserunning, outs, live count.  
- **Soccer:** possession events, shots, tackles, passes, x/y field coordinates.  
- **Tennis/Track:** point outcome, serve speed, lap time, split intervals.

Each event includes:
- Event timestamp and period/inning/quarter  
- Actor (athlete ID) and team  
- Spatial position (if applicable)  
- Event type and outcome  
- Metadata payload (JSON) for extensions

You already captured this accurately in your **Event** table concept.

### How They Turn Data Into a Live Gamecast

**1. Event Processor (Server‑Side)**  
Transforms raw feed into unified event models.  
- For example: NBA “FGM” → `{event_type: "shot_made", x: 25, y: 14, points: 2}`  
- Football “Pass Complete” → `{event_type: "pass_complete", yards: 13, receiver: "Mahomes"}`  

Each sport relies on its **own adapter** but shares downstream logic.

**2. Game State Engine**  
Recomputes current score, possession, player stats, streaks, and win probability.  
It’s the heartbeat of Gamecast—every incoming event mutates the state tree.

**3. Visualization Renderer (Front‑End)**  
Each sport has a *template canvas*:
- NBA/NCAA: Halfcourt SVG with moving shot dots and lines.  
- NFL: 100‑yard grid with live ball marker.  
- Soccer: Field mini‑map with live attack animations.  
- Baseball: Diamond overlay showing base runners.  

Animations are triggered when state deltas appear (e.g., possession swap, new score).

**4. Narrative/Analytics Layer**  
Generates human‑readable updates:  
“LeBron hits a 3‑pointer from 27 feet — Lakers lead 98‑95.”  
Later, machine narratives summarize periods and trends (momentum graphs, win probabilities).

**5. User Sync Layer**  
- WebSockets push JSON deltas to all clients.  
- Redis or Pub/Sub handles scaling.  
- UI subscribes to the game’s channel, applying only event diffs (minimal payloads).

***

### ESPN-Like Design Patterns by Sport

- **NBA/NCAA Basketball:**  
  Fastest update cadence (≈2 sec). Focus on shot location plot + live box score.

- **NFL Football:**  
  Slower cadence (play-based). Emphasizes down‑distance tracker, drive chart, and play sequence timeline.

- **Soccer (Global):**  
  Continuous flow; tracked actions every few seconds via positional data feeds. Emphasis on live HeatMap + possession flow bar.

- **MLB Baseball:**  
  Pitch-by-pitch timeline. Gamecast visualizes strike zone and base runner states.

Each of these sits atop identical logic: events → state updates → UI render.
