# The honest source map

## 1) League of Legends / VALORANT

Riot’s public developer portal is official and well maintained, but it mainly covers game APIs rather than a fully documented public esports product. Community sources consistently note that the LoL esports site uses endpoints that are **not officially supported public esports APIs**. Riot has also publicly said its LoL Esports Data Portal was built with Bayes to support teams, partners, and the community, showing the competitive data ecosystem is real, but it is not the same thing as a simple public plug-and-play esports dev API for your app. ([Riot Developer Portal][4])

**What that means for you:**
Use a Riot provider layer that can ingest:

* official Riot game/ecosystem data where relevant
* wrapped lolesports/valorantesports schedule + standings data where stable
* your own caching and version shielding

## 2) Overwatch

Blizzard is actively running the OWCS ecosystem in 2026 and publishes official season structure and event info. Overwatch also has official hero statistics pages and current competitive ecosystem pages. ([Overwatch][5])

**What that means for you:**
Overwatch can be built around:

* official OWCS schedule/news pages
* official hero/meta data
* normalized team/event results through your provider layer

## 3) CS2

HLTV is the giant in the room for Counter-Strike data and stats, but wrappers around it are explicitly **unofficial** and can get rate-limited or blocked. The popular Node wrapper even warns that abuse can lead to IP bans because of Cloudflare/bot protection. ([HLTV.org][6])

**What that means for you:**
Do **not** build your app around scraping HLTV directly from mobile.
If you use HLTV-derived data at all, wrap it server-side, cache it hard, and treat it as a fragile adapter.

## 4) Dota 2 / Steam-backed titles

Valve’s Steam Web API is official and current, but it is more platform/game-service oriented than a clean consumer-grade esports league feed. ([Valve Developer Community][2])

**What that means for you:**
Dota should be handled with a Valve/Steam adapter plus tournament/result normalization from your secondary providers.

## 5) Cross-title coverage

PandaScore is broad, esports-native, and offers a free plan with 1,000 requests/hour. They specifically position themselves as providing real-time data across titles like LoL, CS2, Dota 2, Valorant, and Overwatch. ([PandaScore][3])

**What that means for you:**
PandaScore is your best **broad fallback / bridge provider** while staying out of paid feeds for now.

## `src/esports/services/esportsRouter.ts`

```ts
import { providers } from '../providers/providerRegistry';
import type { EsportsGame } from '../providers/types';

export async function getBestMatches(game: EsportsGame, status?: Array<'upcoming' | 'live' | 'completed'>) {
  const capable = providers.filter((p) => p.supports(game));

  for (const provider of capable) {
    try {
      const results = await provider.getMatches({ game, status });
      if (results?.length) {
        return {
          provider: provider.name,
          results,
        };
      }
    } catch (err) {
      continue;
    }
  }

  return {
    provider: null,
    results: [],
  };
}
```

# Example: PandaScore adapter

This is the easiest real bridge to start broad.

```ts
import axios from 'axios';
import type { EsportsProvider, EsportsGame } from './types';

const GAME_MAP: Record<EsportsGame, string | null> = {
  lol: 'lol',
  valorant: 'valorant',
  cs2: 'csgo',
  dota2: 'dota2',
  overwatch: 'ow',
};

export class PandaScoreProvider implements EsportsProvider {
  name = 'pandascore';

  supports(game: EsportsGame) {
    return !!GAME_MAP[game];
  }

  async getMatches(args: {
    game: EsportsGame;
    status?: Array<'upcoming' | 'live' | 'completed'>;
    from?: string;
    to?: string;
  }) {
    const gamePath = GAME_MAP[args.game];
    if (!gamePath) return [];

    const statuses = args.status?.length ? args.status.join(',') : 'running,not_started,finished';

    const res = await axios.get(`https://api.pandascore.co/${gamePath}/matches`, {
      headers: {
        Authorization: `Bearer ${process.env.PANDASCORE_API_KEY}`,
      },
      params: {
        'filter[status]': statuses,
        sort: '-begin_at',
        per_page: 50,
      },
      timeout: 12000,
    });

    return (res.data || []).map((m: any) => ({
      id: `pandascore:${m.id}`,
      game: args.game,
      tournamentId: m.league_id ? String(m.league_id) : '',
      tournamentName: m.league?.name || m.serie?.full_name || 'Unknown Tournament',
      stage: m.serie?.name || undefined,
      round: m.name || undefined,
      startTime: m.begin_at,
      status:
        m.status === 'running'
          ? 'live'
          : m.status === 'finished'
          ? 'completed'
          : 'upcoming',
      bestOf: m.number_of_games,
      teams: {
        home: {
          id: String(m.opponents?.[0]?.opponent?.id || ''),
          name: m.opponents?.[0]?.opponent?.name || 'TBD',
          logo: m.opponents?.[0]?.opponent?.image_url || undefined,
        },
        away: {
          id: String(m.opponents?.[1]?.opponent?.id || ''),
          name: m.opponents?.[1]?.opponent?.name || 'TBD',
          logo: m.opponents?.[1]?.opponent?.image_url || undefined,
        },
      },
      scores: {
        home: m.results?.[0]?.score ?? 0,
        away: m.results?.[1]?.score ?? 0,
      },
      rawProvider: this.name,
    }));
  }
}
```
